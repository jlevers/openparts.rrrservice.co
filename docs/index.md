# OpenParts

**Open-source mechanical parts database. Built by mechanics, for mechanics.**

---

You know the drill. You need a brake disc for a '93 Massey Ferguson and you're six tabs deep in dodgy parts websites, cross-referencing OEM numbers against Sparex catalogs, checking dimensions because the aftermarket listing doesn't specify ID/OD, and wondering if that David Brown part is actually the same thing with a different stamp.

OpenParts exists so you don't have to do that anymore.

This is a free, open-source, community-driven parts database. Every entry includes OEM numbers, cross-references, dimensions, and practical notes from people who've actually had the parts in their hands. No paywalls. No dealer logins. No "call for pricing."

## The Three Tenets

### 🔧 Repair

Fix it, don't trash it. Machines are worth rebuilding. A $15 seal kit beats a $400 replacement cylinder every time — if you know the seal kit exists and where to get it.

### 🛡️ Resilience

Dealers close. Parts houses get bought out. Catalogs go offline. Knowledge that lives in one person's head or one company's database is fragile. OpenParts builds a parts knowledge base that outlasts any single source.

### ♻️ Regeneration

Keep machines running. Keep stuff out of landfills. Every tractor that gets rebuilt instead of scrapped is a win. Every part number documented here makes that rebuild a little easier for the next person.

## How to Contribute

OpenParts lives on GitHub. If you've got parts knowledge to share:

1. **Fork the repo** → [jlevers/openparts.rrrservice.co](https://github.com/jlevers/openparts.rrrservice.co)
2. **Add or edit** parts data in Markdown
3. **Submit a pull request** — we'll review and merge

No special tools needed. If you can write a text file, you can contribute. See the **[Contributing Guide](contributing.md)** for details on format and what makes a good entry.

!!! tip "Don't have a GitHub account?"
    If you've got parts info to share but don't want to deal with Git, open an issue on the repo with your data and someone will format it. The knowledge matters more than the workflow.
